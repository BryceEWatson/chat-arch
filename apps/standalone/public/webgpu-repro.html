WebGPU + @huggingface/transformers minimal repro

Loads transformers.js v3.8.1 (matching the project pin) from the same self-hosted /ort-wasm/ directory the viewer worker uses. Calls pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2', { device: 'webgpu', dtype: 'q4f16' }). Logs every step. If this fails, the bug is upstream of our worker wiring (driver / browser / transformers / ORT). If it succeeds, the bug is in our wiring.

What this tests: CDN-loaded transformers.js with its own bundled ORT-web reaching CDN .wasm files. This isolates whether WebGPU + transformers + this hardware works AT ALL, free of any local-build ABI concerns. To test our self-hosted /ort-wasm/ wiring specifically, use the viewer at / → click ANALYZE TOPICS — that path uses local-node_modules transformers + the local ORT it was built against, which is the only ABI-matched pairing for our self-hosted .wasm files.
